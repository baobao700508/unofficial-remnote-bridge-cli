# Exporting Notes

You can export your notes and flashcards to a variety of file formats to share them with other applications or people who don't use RemNote.

If you're trying to let someone else look at or work on your notes, consider [sharing them](https://help.remnote.com/en/articles/6030805-sharing-rem) or [collaborating on a knowledge base](https://help.remnote.com/en/articles/6689499-collaboration-and-shared-knowledge-bases) instead – these methods are usually easier for both of you!

# Exporting a document

In the document you'd like to export, click the **…** menu in the upper-right and select _Export_. Select an export type and choose any options required; with most formats, you'll be able to see a preview of the effect the options have on your exported content. When it looks right, click the blue download button at the bottom and choose where to save your exported file.

## Exporting as PDF

In addition to exporting your notes in various file formats, you can convert any RemNote document to a PDF using your system’s print functionality. This can be helpful if you want a PDF version for sharing, archiving, or offline viewing.

To print a RemNote document as a PDF:

1.  Open the document you want to print.
    
2.  Click the **…** menu in the upper-right corner and select _Print_.
    
3.  In the print dialog that appears, choose **Save as PDF** (or a similar option, depending on your browser and operating system) as your printer destination.
    
4.  Adjust the print settings as needed (such as layout, margins, or which pages to include).
    
5.  Click **Save** or **Print** to generate and download the PDF file.

# Exporting your entire knowledge base

Go to _Settings > Export_.

As with exporting a single document, you can select a file format and any options for that file format, then click the blue _Export_ button to export everything.

The most common reasons to export your whole knowledge base are to [make a backup](https://help.remnote.com/en/articles/6301627-remnote-backups) and to [move it to or merge it into another knowledge base](https://help.remnote.com/en/articles/7868482-moving-content-between-knowledge-bases); see those articles for details.

# What format should I choose?

When exporting, there are several file formats available. Choosing the right one requires knowing what you will do with it after. These are listed roughly in order of fidelity, or information retained. Usually you should pick the format closest to the top of the list that the application you're trying to transfer data to understands.

-   **RemNote (Complete):** RemNote native format. It can be imported back into RemNote without losing any information. Note, however, that it does not currently include images or PDFs. This media is maintained on RemNote's servers, so this is usually only an issue [if you're exporting from a local knowledge base](https://help.remnote.com/en/articles/6758834-how-do-i-move-content-from-a-local-kb-to-a-synced-kb); see there for more information.
    
-   **OPML**: The [open interchange format for outliners](https://en.wikipedia.org/wiki/OPML), a form of XML.
    
-   **Anki (Flashcards Only):** An .apkg file ready to [import](https://help.remnote.com/en/articles/6751471-importing-from-anki) into [Anki](https://apps.ankiweb.net). Note that Rems which have no flashcards will be ignored in this format!
    
    -   Additionally, please note that exported Anki flashcards will intentionally include their parent/hierarchical structure to preserve the necessary context from RemNote.
        
    
-   **HTML:** Like text, but with text formatting rendered in HTML, the formatting language for websites.
    
-   **Markdown**: Like text, but with text formatting rendered in [Markdown](https://en.wikipedia.org/wiki/Markdown).
    
-   **Text:** A plain-text version of your notes indented based on the outline hierarchy.
    
-   **RemNote (Without Text of Notes):** Use this format only if requested by RemNote support.
    

If you only need to copy a small amount of content from one document, it's often easier to simply **copy and paste it** into the application you'd like to move it to, rather than exporting it at all. This often results in better formatting as well, since the other application can tell RemNote what kind of formatting instructions it understands in this case.
